# Basic Regex

Базовий Regex, реалізований через скінченний автомат в рамках лабораторної роботи з дискретної математики.

## Підтримувані оператори

| Оператор | Опис |
|---|---|
| `.` | Будь-який один символ |
| `*` | Попередній символ нуль або більше разів |
| `+` | Попередній символ один або більше разів |

---

## Пояснення операторів

### `.` — будь-який один символ

Патерн:
```python
"a.c"
```

Приклади:
```python
"abc"  -> True
"a5c"  -> True
"ac"   -> False   # '.' вимагає рівно один символ
"axc"  -> True
```

---

### `*` — попередній символ нуль або більше разів

Патерн:
```python
"a*b"
```

Приклади:
```python
"b"     -> True   # символ 'a' відсутній
"ab"    -> True   # один раз
"aaab"  -> True   # декілька разів
```

---

### `+` — попередній символ один або більше разів

Патерн:
```python
"a+b"
```

Приклади:
```python
""      -> False  # хоча б один 'a' є обов’язковим
"b"     -> False
"ab"    -> True
"aab"   -> True
```

---

## Опис проєкту

У межах цього проєкту було реалізовано скінченний автомат, який перевіряє, чи відповідає введена стрічка заданому патерну.

Програма підтримує:

- великі та малі літери англійського алфавіту (`A-Z`, `a-z`)
- цифри (`0-9`)
- базові оператори (`.`, `*`, `+`)
- коректну обробку edge-cases

---

## Приклад використання

### 1. Створіть regex-патерн

```python
regex_pattern = "a*4.+hi"
```

### 2. Передайте патерн у FSM

```python
regex_compiled = RegexFSM(regex_pattern)
```

### 3. Перевіряйте стрічки на відповідність патерну

```python
print(regex_compiled.check_string("aaaaaa4uhi"))
```

Результат:
```python
True
```

Програма виводить у термінал `True` або `False` залежно від того, чи відповідає стрічка заданому патерну.

## Пояснення реалізації

Загалом більшість реалізації виявилась доволі легкою.

Найскладнішою частиною було написання функції `get_epsilon_closure`, яка реалізовує епселон (деколи лямбда позначають ше) переходи між станами автомата.

Реалізував через ітеративне розширення множини станів:
алгоритм продовжує шукати нові доступні стани доти, доки існують переходи через оператори `*` та `+`, які не споживають символи з вхідної стрічки.
